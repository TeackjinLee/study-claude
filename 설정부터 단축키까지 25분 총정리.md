# Claude Code 입문 완벽 가이드

> 이 페이지는 **Claude Code 입문 완벽 가이드** 영상의 핵심 내용을 정리한 치트시트입니다.
> 초기 설정부터 필수 명령어까지, 이 한 장이면 충분합니다.

---

# 1. 초기 설정 & claude.md

## 🚀 시작하기

### 1️⃣ 프로젝트 루트 디렉토리에서 Claude Code 실행

Claude는 현재 디렉토리를 기준으로 파일을 탐색합니다.

프로젝트 루트가 아닌 곳에서 실행하면 프로젝트 구조를 제대로 파악하지 못할 수 있습니다.

### 2️⃣ `/init` 명령어로 `claude.md` 자동 생성

```bash
/init
```

Claude가 프로젝트를 분석하여 프로젝트 설명서(`claude.md`)를 자동 생성합니다.

---

## 📂 claude.md 계층 구조

```text
~/.claude/claude.md
└─ 글로벌 설정 (모든 프로젝트 공통)

프로젝트루트/claude.md
└─ 프로젝트 전용 설정
```

### 활용 예시

#### 글로벌 설정

```markdown
항상 한국어로 답변해줘.
```

#### 프로젝트 설정

```markdown
Spring Boot + Nexacro 기반 프로젝트
Controller → Service → DAO 구조 사용
```

  → staging 환경 배포

`claude.md`를 Git 저장소에 커밋하면 팀 전체가 동일한 규칙을 사용할 수 있습니다.

# 2. 키보드 단축키 & 네비게이션

## ⌨️ 핵심 단축키

### Shift + Tab

Plan Mode ↔ Accept Mode 전환

#### Plan Mode

* 계획만 수립
* 실제 실행 안 함

#### Accept Mode

* 계획 수립
* 실제 실행까지 수행

### 권장 방식

새 작업은 항상 Plan Mode로 시작

```text
Plan
 ↓
검토
 ↓
Accept
```

---

## 🧠 왜 Plan Mode를 사용해야 할까?

### 1. 토큰 최적화

불필요한 코드 생성을 줄여줍니다.

잘못된 방향으로 생성된 수천 줄의 코드보다 계획을 먼저 검토하는 것이 훨씬 효율적입니다.

---

### 2. 컨텍스트 보존

긴 대화가 지속되면 컨텍스트가 압축되거나 손실됩니다.

Plan Mode는 적은 토큰으로 방향성을 먼저 정리할 수 있습니다.

---

### 3. 복잡한 작업 분해

예시:

```text
Step 1
Step 2
Step 3
```

단계별 승인 방식으로 실패율을 낮출 수 있습니다.

---

### 4. 수정 범위 통제

Claude가 어떤 파일을 수정할지 미리 확인 가능합니다.

사이드 이펙트를 줄일 수 있습니다.

---

### 5. 비용 절감

API 사용 시 토큰 비용 절약 효과가 큽니다.

커뮤니티 기준 체감상

* 30%
* 50%

정도의 절감 효과가 보고되고 있습니다.

# 3. 필수 슬래시 명령어

## 🧹 컨텍스트 관리

### `/clear`

컨텍스트 완전 초기화

새 작업 시작 시 사용

```bash
/clear
```

---

### `/context`

현재 토큰 사용량 확인

```bash
/context
```

사용량이 80% 이상이면

* `/clear`
* `/compact`

사용 권장

---

### `/compact`

컨텍스트 압축

```bash
/compact
```

#### 차이점

```text
/clear
= 완전 초기화

/compact
= 맥락 유지 + 압축
```

---
